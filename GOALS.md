# liber

I've always wanted to have my own web browser.

Most of this functionality could be provided by a really aggressive browser
extension, I think, so it's fine to just do that (within chrome). But if
that seems too hard we can also write our own browser app based on webkit
views. This would mostly be for personal study and browsing.

This should be for macos and ios.

Some ideas to start with:

keyboard shortcuts for literally everything (for now, just do basic
navigation). Flipping through information at multiple granularities should
be fast and fluent. Lots of indices and cross references should be cached to
make this easier

try to discourage searching, or typing in URLs. Things can be added to
curated lists of pages or sites instead. External URLs should be discouraged
unless they link between specific known sites.

try to discourage aggregators like reddit and hacker news. These should be
re-curated by LLMs, comment sections and junk hidden, and remaining new
items categorized.

knowledge should be acquired intentionally rather than by random chance.
That is, web sites and pages should be broken down by categories.

When pages are read, they should automatically be curated, filed, and
flagged for study, potential rereading, entertainment, or discarded. There
should be an actively maintained (possibly by an LLM) reading log

Browsing should be serendipitous in the positive, curated direction. It
should be possible to access "displays" / "staff picks" of articles, sites,
and resources. These should be somewhat informed by my own interests but
mostly by an actual bar of quality and thoughtfulness.

news / timely interest topics and stories should be flagged as such and
organized together. Almost all topics and resources should not be news, but
should be grouped by long-lasting topic areas (within e.g. technology,
science, etc).

when looking around, everything should load rather slowly, with
opportunities to browse other nearby (in idea-space things) readily
available. But once dived into a thing it should be a bit difficult to
extricate yourself and go somewhere else. In particular there shouldn't be
a tab bar at the top all the time.

all distractions should be removed whenever possible, i.e., the default
should be reader mode (or a pdf viewer) with nothing else on screen, and
only keyboard shortcuts available. But the rest of the UI surface should
return as needed.

it should be possible and, indeed, mandatory to take active notes while
reading. It's better to take notes on paper, so it should be easy to hold
these up to the camera and attach/OCR them to the thing being read.

when relevant there should be some follow-up assessments for individual
articles, groups of articles, etc, to ensure good learning and following up
is happening. These could be self-imposed or LLM-generated.

if something is hard there should be some mild (but not insurmountable)
pressure to continue. But it's always ok to skip challenges and just keep
looking around. There's no penalty for putting things away except spending
time looking for the next thing.

there should be time tracking for everything that goes through the app as
well, to look at usage and interest by topic. Again this is for personal use
not to impose a work schedule with rest breaks etc.
